# 1. プロジェクト概要
# 目的
・好奇心を刺激し、実践的な情報を提供するによって移動のハードルをさげ、多くの人が多様な価値観や文化に触れることで人生の可能性を広げられる旅行情報のプラットフォームを創り上げる。
・ユーザーが一定程度増えた先には、観光産業に依存した都市の課題を解決を支援するプラットフォームに進化させる。

# MVPの価値
・都市を検索し、都市ごとの概要と関連情報（動画、ブログ、記事）を見られること。
・旅の経費を登録し、管理ができること。

# 将来像
・YouTube以外にWikipedia、ブログ、SNS、宿泊価格、季節情報も統合したい。
・旅行者と観光地の現地の人を結びつけることで観光地の特に閑散期の経済を支えるような収入源を創出したい。
・ユーザーからへ課金するビジネスモデルではなく、ユーザー情報や蓄積されたトラフィック情報を価値の源泉とするようなビジネスモデルとしたい。
・例えば、ユーザーに対する人材紹介や金融、保険商品の仲介、他のプラットフォームではないセグメンテーションによるサービス、物販。

# 4. 制約条件
## 業務ルール・制約メモ
# 制約
- 初期のUIはStreamlitを使うが、やりたいことが実現できなくなったら別の言語に移行する。
- データソース：初期はGoogle Sheetsだが、データ量が増えてきたらデータベースを使う
- YouTube APIは呼び出し回数を最小化したい
- 外部API：Youtube、OpenaiやGoogleGeminiなどの生成AI系、WISEなどの資金の出入りを自動連係できるような金融系
- APIコスト：最小限にしたい。
- レート制限
- データ更新頻度
- 手入力するデータ
- 自動取得するデータ:youtube動画、wikipedia、ブログ、
- 運用体制：1人
- 予算や開発人数：開発はすべて自分1人で開発する。
